Installation

Create a GitHub account

Start by signing up for a (free) account on GitHub. We will be using GitHub to teach version control with the Git version control system.

Note: A common confusion among students new to Git is that Git and GitHub are not the same thing. Git is a version control system that we will be teaching, and is software that you run on your computer. GitHub is a website for hosting Git repositories publicly.

Installation help

If you need help with installation, please email the instructors at info@software-carpentry.org.

What to install

To complete the entire workshop you need several things: a Bash shell, git, a code editor (though any plain text editor will work in a pinch), and a scientific Python installation that includes the IPython Notebook, NumPy, and matplotlib.

In the long run it is better for the student to install all of the software on their own computer. But even if you plan to install things yourself please download VirtualBox and the virtual machine as a backup (see below). It can sometimes be very difficult to get everything set up correctly—software installation remains, sadly, one of the most difficult aspects of scientific computing, and the virtual machine works well as a last resort for the sake of following the class. After the class or during breaks the instructors and helpers can help you troubleshoot installation issues on your machine.

Bash

Mac

The default shell in Mac OSX is bash.

Windows

Install Git Bash following the instructions here (Git Bash gives you both the Git version control system and the Bash shell in a single package).

Linux

The default shell is usually bash but if not you can get to bash by opening a terminal and typing bash.

Git

Mac

Install Xcode and the command line tools (from the Download preferences pane) or install just git.

Windows

Install Git Bash following the instructions here.

Linux

If git is not already available on your machine you can try to install it via your distro's package manager (e.g. on Debian/Ubuntu run sudo apt-get install git).

Code Editor

If you already have a favorite text/code editor please use whatever you are comfortable with.

Sublime Text is an excellent editor that works on Mac, Windows, and most Linux distributions. It's paid software, but it has an unlimited trial period. There are also some other, platform specific options:

Mac

We recommend Text Wrangler, or Text Mate 2.

Windows

Notepad++ is a popular free code editor for Windows.

Linux

Kate and gedit are options for Linux users.

Python

Mac & Linux

We recommend the all-in-one scientific Python installer Anaconda. Installation requires using the shell and if you aren't comfortable doing the installation yourself just download the installer and we'll help you at the boot camp.

For other options check the Python4Astronomers page on installing scientific Python.

Windows

For Windows we recommend the Enthought Canopy distribution since it seems to work well with Git Bash.

Virtual Machine

To give everyone a consistent, pre-configured environment we provide a Linux virtual machine. However, as it is better for long-term continued use of the software you will learn at this bootcamp it is better to have everything working natively on your machine. However, the virtual machine is a good last resort to get you up and running at the bootcamp. Install VirtualBox and download this virtual machine image (alternate download). Load the VM into VirtualBox by doing Import Appliance and loading the .ova file. The default username/password to log into the VM is swc/swc.

Testing your Installation

Once you've gone through the installation above, please download (save to your disk) this Python script and execute it as per the instructions contained in it (at the top). It runs a check on your Python version. Next, download this script and run it in the same way as the previous one. It performs a set of sanity checks on your system to ensure that things are reasonably well installed, and will provide diagnostics information along with instructions on what to do if anything is missing.

How to get a command line and run the scripts:

  • On OSX run the Terminal application.
  • On Windows, once you have installed Git Bash start the bash terminal from the desktop shortcut.
  • Before running the scripts, change directories to the path you downloaded them to (for example on OSX this might default to /Users/<your_username>/Downloads). Either way pay close attention to where you downloaded the files. To change directories, at the prompt enter: cd /Full/Path/To/Downloads, but using the actual directory path beginning with /. If you're not sure about this please ask on the mailing list.
  • Follow the instructions the script prints at the end (if any).
